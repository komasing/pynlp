Praktikumi eesmärgiks on seada endale üles töökeskkond edasiste praktikumiülesannete lahendamiseks ning tutvuda kasutatavate töövahenditega (Python, Jupyter Notebook, Anaconda). Püstitatud ülesannete lahendamine aitab mõista töövahendite kasulikkust ja tunda end nende kasutamisel mugavalt.

Intro Pythoni virtuaalkeskkondadesse

Maailma päästmiseks piisab harva ühest Pythoni projektist. Mitme projektiga tekivad aga sageli versioonikonfliktid - enim Pythonite ja nende standardteekide ning süntaksi osas.

Ehkki neid on tihti võimalik erinevate Pythoni versioonide installimisega likvideerida (nt käsureal python, python3, python2.7 jms), läheb raskeks, kui ühel Pythoni versioonil konkureerivad projektid sama välise teegi erinevate versioonidega (nt lxml X.Y, Z.W).

Projektides kipuvad jääma esialgsed teegi- ja arhitektuurivalikud pikalt püsima (üldiselt mugavusest, harvem vajadusest) ning sellest lähtuvalt ei ole iga projekti iga teek viimasel võimalikul versioonil, põhjustades vananenud teekide vahel konflikte.

Pythoni kiire prototüüpimise disaini kohaselt installitakse teegid globaalselt, et iga uue projekti korral ei peaks hakkama uuesti projektiväliseid sõltuvusi (dependence’eid) süsteemist/veebist otsima. Kuigi tegu on olulise lihtsustusega, läheb see teekide erinevate versioonidega kiiresti katki.

Vanilla (“tavalises”) Pythonis installitakse uusi teeke harilikult pip käsu abil (pip install numpy). Seega võime projektide vahel jooksutada pip install numpy==1.2.3 ja pip install numpy==1.1.5. Konfliktsete teekide arvu tõusuga võib aga arvepidamine käest minna ning lisaks peab pip-i korral igal installimisel kompileerima, mis tähendab tarbetut ajakulu.

Appi tõttavad Pythoni virtuaalkeskkonnad, mis kopeerivad kogu Pythoni rakenduse kasutajale kättesaadavasse kataloogi. See tähendab, et mitte ainult ei ole meil nüüd teist Pythonit, kuhu võime installida teise projekti teekide versioonid, vaid saame ise hallata oma teeke ka juhul, kui meil puudub kirjutamisõigus lähte-Pythoni kataloogile (serverid, korporatiivne poliitika turvariskide ennetamiseks vms).

Käesolevas aines soovitame vabavaralist Continuumi Anacondat, mis tuleb vaikimisi olulisemate teaduslike teekidega ning kus on palju teeke ette kompileeritud.

Valik sagedasemaid käske:

Uue keskkonna loomine:

conda create --name keskkonna_nimi [python=3.5] [installitavad paketid tühikuga eraldatult]

Loodud keskkonna aktiveerimine:

Linux, OSX: source activate keskkonna_nimi

WIN: activate keskkonna_nimi

Loodud keskkonna deaktiveerimine:

Linux, OSX: source deactivate keskkonna_nimi

WIN: deactivate keskkonna_nimi

Uue teegi installimine aktiivsesse keskkonda (vaikekeskkond root):

conda install teegi_nimi[==versioon] # kui teek on Anaconda repositooriumis

pip install teegi_nimi[==versioon] # kui teeki ei ole Anacondas, aga on PyPA-s

Google + installimine vastavalt teegi docile muul juhul

Loodud keskkondade loetelu:  conda env list

Jupyteri installimine ja condaga sidumine.

Jupyter on Anacondaga kaasas. Virtuaalkeskkonna loomisel ning minicondasse tõmbamisel saab läbi conda tõmmata:

conda install jupyter

Läbi Conda tõmmatud Jupyter juba toetab Anaconda keskkondi.

Jupyter Notebookist

Jupyter Notebook on graafiline rakendus, mis võimaldab erinevates programmeerimiskeeltes interaktiivselt programmeerida ning testida erinevaid ridu ja koodiplokke, talletades eelmiste plokkide muutujate väärtusi (sh imporditud teegid, defineeritud funktsioonid jms). Lisaks on võimalik koodiridade vahel kirjutada dokumentatsioonina nt HTML-i ja LaTeX-it ning visualiseerida tabeleid, graafikuid ja pilte. Dokumente käsitletakse märkmikena (notebook) ning neid saab teistega jagada ning eksportida erinevatesse failiformaatidesse.

Rakenduse käivitamiseks liigu käsureal soovitud kausta (mis saab rakenduse juureks, kust failisüsteemis kõrgemale ei anna enam liikuda) ning jooksuta

jupyter notebook

mille tulemusel avaneb veebilehitsejas uus tab (pilt 1).
